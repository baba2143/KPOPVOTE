## 🖥️ 管理画面のWeb UI設計と技術仕様

管理画面は、モバイルアプリと同じFirebaseエコシステム上に**Webベース**で構築されます。

### 1. Web技術選定

| 項目 | 推奨技術 | 理由 |
| :--- | :--- | :--- |
| **フロントエンド** | **React / Vue.js** | モジュラリティが高く、複雑な管理画面UIの構築に適している。 |
| **UIライブラリ** | **Material UI (MUI) / Ant Design** | フォームやテーブルなど、管理画面に必要なコンポーネントを迅速に実装可能。 |
| **ホスティング** | **Firebase Hosting** | Firebaseとのネイティブな連携と、簡単なデプロイ・運用を実現。 |
| **バックエンド接続** | **Cloud Functions API** | 管理者認証を必須とし、セキュリティを担保した上でデータ操作を実行。 |

### 2. グローバルナビゲーション（左側固定）

画面の左側に固定ナビゲーションバーを設置し、主要機能へのアクセスを提供します。

* **ダッシュボード** (Home)
* **コンテンツ管理** (独自投票、コミュニティ)
* **マスター管理** (アイドル、外部アプリ)
* **ユーザー管理**
* **システムログ**

### 3. 主要画面の構成案

管理画面は、直感的なデータ参照と効率的な作業に重点を置いて設計します。

| 画面名 | 構成要素と必要な機能 | Web UIのポイント |
| :--- | :--- | :--- |
| **A. ダッシュボード** | 新規/アクティブユーザー数、タスク登録総数、エラーログサマリーの**グラフ表示**。 | **データ視覚化**（グラフコンポーネント）を多用し、サービスの健全性を一目で把握。 |
| **B. 独自投票管理** | 現在の投票のステータス、新規投票の作成フォーム、候補者と票数のリアルタイムランキング表示。 | フォーム入力とデータテーブル（表）のコンポーネントを主に利用。**「新規作成」ボタンを最上位に配置。** |
| **C. マスターデータ管理** | アイドル/メンバーリスト、外部アプリマスターの登録・編集・削除機能。フィルタリング、検索機能。 | フィルタリング機能付きの**データテーブルコンポーネント**を使用し、大量のデータを効率的に管理。 |
| **D. コミュニティ監視** | 不適切報告のあった投稿を優先表示。投稿の確認、**削除ボタン**。 | モデレーターが迅速に判断できるよう、**報告された投稿**をリストの最上部に強調表示。 |
| **E. ユーザー管理** | 全ユーザーの検索機能。特定ユーザーの詳細ページ（推し設定、ポイント残高、タスク登録状況）の参照。 | ユーザーIDや名前での**強力な検索バー**と、詳細データを表示するためのモーダルウィンドウ（ポップアップ）。 |
